Bug Critico: Los Datos del Lanzamiento se Perdia

Problema: El log mostraba power=2.93 pero el impulso era (0, 0, 0).

Causa: AimingState.exit() llamaba a throw_controller.stop_aiming() que reseteaba TODOS los datos del gesto (potencia, direccion, curva) a cero. Cuando ThrowingState intentaba leer los datos, ya estaban reseteados.

Secuencia del bug:
  1. AimingState detecta RELEASED → calcula potencia = 2.93
  2. AimingState llama transition_to(THROWING)
  3. StateMachine llama AimingState.exit() → stop_aiming() → datos = 0
  4. StateMachine llama ThrowingState.enter()
  5. ThrowingState llama get_throw_data() → devuelve {power: 0, direction: 0, curve: 0}
  6. Impulso = (0, 0, 0) → la bocha no se mueve

Fix: AimingState.exit() ya NO llama a stop_aiming(). ThrowingState.exit() lo llama despues de usar los datos.

Gesto Simplificado: "Pull & Release"

El gesto anterior requeria 3 fases (click → arrastrar abajo → arrastrar arriba → soltar). Era demasiado complejo y poco intuitivo.

Nuevo gesto (estilo tirachinas):
  1. Click y mantené presionado en cualquier parte de la pantalla
  2. Arrastra el mouse en cualquier direccion (la distancia = potencia)
  3. Soltá el boton → la bocha se lanza

Como funciona:
  • Distancia arrastrada → potencia del lanzamiento (0 a max_throw_power)
  • Direccion del arrastre → siempre lanza hacia adelante (-X)
  • Componente lateral del arrastre → efecto de curva (izquierda/derecha)

Configuracion:
  • min_drag_distance = 20px → arrastre minimo para lanzar
  • max_drag_distance = 400px → arrastre para potencia maxima

Como Probar

  1. Corre la escena (F6)
  2. Presioná A o D para mover la bocha lateralmente
  3. Hacé click izquierdo en cualquier lado → entra a AIMING
  4. Mantené presionado el boton y arrastrá el mouse unos centimetros
  5. Soltá → la bocha se lanza hacia adelante
