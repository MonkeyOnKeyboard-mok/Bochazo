🎯 IA de Tiro - Arbol de Decisiones

Resumen

La IA saltea todo el sistema de gestos del jugador y directamente genera los parametros que ThrowFlight necesita para lanzar la bocha. Se entrena un Arbol de Decisiones simulando tiros al azar y evaluando si el resultado final esta mas cerca del bochin que las bochas existentes.

Principio clave: La IA NO dibuja. La IA NO usa GestureTracker/GesturePower/ThrowAim. La IA genera directamente los parametros de ThrowFlight.launch() y ThrowFlight.launch_straight().

4.2 Simplificacion: Las 2 bochas mas cercanas

Para limitar la complejidad del arbol, solo consideramos las 2 bochas mas cercanas al bochin:

  • nearest_ally_1: bocha aliada mas cercana al bochin
  • nearest_ally_2: segunda bocha aliada mas cercana (o null)
  • nearest_enemy_1: bocha enemiga mas cercana al bochin
  • nearest_enemy_2: segunda bocha enemiga mas cercana (o null)

De cada una solo guardamos dist_to_bochin y angle_relative.

5. Arbol de Decisiones

5.1 Estructura del nodo

class_name AITreeNode
extends Resource

# Nodo de pregunta (split)
var feature: String = ""        # ej: "bochin_dist", "nearest_enemy_dist"
var threshold: float = 0.0     # valor de corte
var left: AITreeNode = null     # feature < threshold
var right: AITreeNode = null    # feature >= threshold

# Nodo hoja (decision)
var params: AIThrowParams = null  # si es hoja, tiene los parametros
var is_leaf: bool = false

5.2 Features del arbol

El arbol hace preguntas sobre el contexto. Cada nodo interno pregunta:

if context[feature] < threshold:
    go left
else:
    go right

Features posibles con sus discretizaciones:

FeatureValoresEjemplo pregunta
bochin_dist_norm0.0 - 1.0"¿El bochin esta cerca?" → threshold 0.5
bochin_angle-1.0 - 1.0"¿El bochin esta a la izquierda?"
nearest_enemy_dist0.0 - 1.0"¿Hay un enemigo cerca del bochin?"
offense_need0, 1, 2"¿Estamos perdiendo?"
court_type0 - 4"¿Es cancha de arena?"

5.3 Hojas del arbol = AIThrowParams

Cada hoja contiene:

  • power: fuerza del lanzamiento (0-1)
  • angle_offset: angulo de desviacion respecto al bochin (radianes)
  • curve_intensity: 0 = recto, >0 = curva con waypoints laterales
  • curve_side: -1 izquierda, +1 derecha, 0 recto

A partir de estos valores, AIThrowBrain calcula los waypoints 3D y el direction final.

6. Entrenamiento

6.1 Algoritmo

  1. Generar contexto aleatorio: posicion del bochin, cancha, bochas existentes, stats del personaje
  2. Generar parametros aleatorios: power, angle_offset, curve_intensity, curve_side
  3. Simular el tiro: lanzar la bola con estos parametros en la fisica de Godot
  4. Esperar a que se detenga: medir distancia final al bochin
  5. Calcular reward:
    • Excelente: Distancia < bocha aliada mas cercana → reward alto
    • Bueno: Distancia < bocha enemiga mas cercana → reward medio
    • Malo: Distancia > bocha enemiga mas cercana → reward bajo
  6. Construir arbol: Dividir los datos usando el feature y threshold que mejor separa los buenos resultados de los malos (CART algorithm)
  7. Podar: Limitar profundidad maxima (5-7 niveles)
  8. Guardar: Serializar el arbol como .tres resource

6.2 Funcion de Reward

func calculate_reward(final_pos, bochin_pos, ally_dists, enemy_dists) -> float:
    var dist_to_bochin = final_pos.distance_to(bochin_pos)
    var best_ally = ally_dists.min() if ally_dists.size() > 0 else INF
    var best_enemy = enemy_dists.min() if enemy_dists.size() > 0 else INF
    
    var score = 0.0
    
    # Bonus por estar cerca del bochin
    score += clampf(1.0 - dist_to_bochin / 30.0, 0.0, 1.0)
    
    # Bonus por superar a la mejor bocha aliada
    if dist_to_bochin < best_ally:
        score += 0.5
    
    # Bonus por superar a la mejor bocha enemiga
    if dist_to_bochin < best_enemy:
        score += 0.3
    
    # Penalty por quedar muy lejos
    if dist_to_bochin > 10.0:
        score -= 0.3
    
    return score

6.3 Generacion de Waypoints para Simulacion

Dado un angle_offset y curve_intensity, la IA genera los waypoints 3D:

func generate_waypoints(ball_pos, bochin_pos, angle_offset, curve_intensity, curve_side):
    var direction = (bochin_pos - ball_pos).normalized()
    direction = direction.rotated(Vector3.UP, angle_offset)
    
    if curve_intensity < 0.05:
        return []  # tiro recto
    
    var dist = ball_pos.distance_to(bochin_pos)
    var wp_count = int(3 + curve_intensity * 5)
    var waypoints = PackedVector3Array()
    var right = direction.cross(Vector3.UP).normalized()
    
    for i in range(wp_count):
        var t = float(i) / float(wp_count - 1)
        var pos = ball_pos + direction * dist * t
        # Curva senoidal: pico en el medio
        var lateral = sin(t * PI) * curve_intensity * curve_side * dist * 0.15
        pos += right * lateral
        pos.y = ball_pos.y
        waypoints.append(pos)
    
    return waypoints

9. Integracion con ThrowSystem Existente

9.1 AIPlayer - Componente nuevo

class_name AIPlayer
extends Node

@export var stats: PlayerThrowStats
@export var ball: BocceBall

var brain: AIThrowBrain
var flight: ThrowFlight

func take_turn(bochin_pos: Vector3, court_type: int, bochas: Array):
    var context = AIContext.gather(bochin_pos, court_type, bochas, stats, ball.global_position)
    var params = brain.decide(context)
    
    # Aplicar stats del personaje
    flight.efecto = stats.efecto
    flight.precision = stats.precision
    flight.control = stats.control
    flight.max_force = stats.potencia
    flight.ball = ball
    
    if params.is_straight or params.waypoints.size() < 2:
        flight.launch_straight(params.power, params.direction)
    else:
        flight.launch(params.power, params.direction, params.waypoints)

9.2 El flujo completo en juego

Turno IA:
  1. AIContext.gather() → recolecta datos del juego
  2. AIThrowBrain.decide(context) → devuelve AIThrowParams
  3. AIThrowParams → ThrowFlight.launch() → bola vuela
  4. Esperar a que la bola se detenga (señal stopped_moving)
  5. Evaluar resultado (para entrenamiento futuro)

14. Detalle de Archivo: ai_tree_node.gd

class_name AITreeNode
extends Resource

@export var feature: String = ""
@export var threshold: float = 0.0
@export var is_leaf: bool = false
@export var params: AIThrowParams
@export var left: AITreeNode
@export var right: AITreeNode

func decide(context: AIContext) -> AIThrowParams:
    if is_leaf:
        return params
    var value = context.get(feature)
    if value < threshold:
        return left.decide(context) if left else params
    else:
        return right.decide(context) if right else params

15.2 Flujo de Entrenamiento

1. Abrir training_AI.tscn en Godot
2. Seleccionar dificultad en el dropdown
3. Click "Train"
4. La escena corre simulaciones a 3x velocidad:
   a. Genera contexto aleatorio (posicion bochin, tipo cancha, stats)
   b. Genera parametros aleatorios de lanzamiento
   c. Lanza TrainingBall con impulso + ThrowFlight
   d. Espera a que la bola se detenga
   e. Calcula reward (cercania al bochin)
   f. Registra (contexto, params, reward)
5. Al terminar, construye arbol CART con AITrainer
6. Click "Save" → guarda ai_tree_diff_N.tres en resources/ai_config/

15.3 Flujo de Test

1. Abrir test_AI.tscn en Godot
2. Seleccionar dificultad (carga el arbol correspondiente)
3. Click "AI Throw"
4. La IA:
   a. Lee contexto del juego (posicion bochin, bochas existentes)
   b. Evalua el arbol de decisiones
   c. Aplica noise segun dificultad
   d. Ejecuta ThrowFlight.launch() con los parametros
5. Click "Reset" para limpiar y lanzar de nuevo
Fisica de Godot: La simulacion de entrenamiento DEBE usar la misma fisica que el juego real. Esto significa que debemos lanzar RigidBody3D reales en una escena de simulacion y esperar a que se detengan. No podemos predecir el resultado solo con matematicas porque las colisiones con paredes y otras bochas son fundamentales.
Optimizacion: Podemos pre-entrenar arboles para cada combinacion de personaje+cancha y guardarlos como .tres. En runtime solo cargamos el arbol correspondiente.
Alternativa simplificada: Si el entrenamiento es muy lento, podemos empezar con un arbol hardcodeado basado en las estrategias de la tabla en seccion 8, y refinarlo con simulacion despues.
