Bocha.tscn — Documentación Funcional

Escena principal de la bocha para el juego Bochazo (Bocce Argentino 3D). Godot 4.6, arquitectura por composición.

1. Arquitectura General

La escena Bocha.tscn sigue un patrón de composición sobre herencia. El nodo raíz (Bocha) actúa como orquestador; cada hijo es un componente autocontenido con responsabilidad única.

┌─────────────────────────────────────────────────┐ │ Bocha (Node3D) │ │ Nodo Raíz / Orquestador │ │ │ │ ┌──────────┐ ┌──────────────┐ ┌────────────┐ │ │ │Physics │ │ StateMachine │ │ Position │ │ │ │Body │ │ │ │Controller │ │ │ │(Rigid) │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ ├Mesh │ │ ├Positioning │ │ │ │ │ │ ├Collision│ │ ├Aiming │ │ │ │ │ │ │ │ ├Throwing │ │ │ │ │ │ │ │ └Rolling │ │ │ │ │ └──────────┘ └──────────────┘ └────────────┘ │ │ │ │ ┌──────────────┐ ┌──────────────┐ ┌─────────┐ │ │ │ThrowController│ │BochaAnimator │ │VisualFx │ │ │ │ │ │ │ │ │ │ │ │ • Drag 2 fases│ │ • Mesh offset│ │ •Trail │ │ │ │ • Raycast │ │ • Color shift│ │ •Decal │ │ │ │ • Power calc │ │ • Arrow dir │ │ │ │ │ └──────────────┘ └──────────────┘ └─────────┘ │ │ │ │ ┌──────────────┐ │ │ │InputHandler │ │ │ │ • Raycast │ │ │ │ • Click check│ │ │ └──────────────┘ │ └──────────────────────────────────────────────────┘
Principio clave: El nodo Bocha solo cachea referencias en _ready() y las inyecta en los hijos. Cada hijo opera de forma autónoma y se comunica a través de la referencia compartida a bocha_root.

2. Árbol de Nodos

Bocha (Node3D) │ ├── PhysicsBody (RigidBody3D) │ ├── MeshInstance3D ← Modelo 3D de la bocha (bocha_azul.obj) │ └── CollisionShape3D ← SphereShape3D (radio = 1.0) │ └── PhysicsMaterial: friction=0.3, bounce=0.2 │ ├── StateMachine (Node) ← Máquina de estados │ ├── PositionController (Node) ← Movimiento lateral A/D ├── ThrowController (Node) ← Gesto de lanzamiento (drag 2 fases) ├── InputHandler (Node) ← Raycast de validación de click ├── BochaAnimator (Node) ← Feedback visual (offset, color, flecha) └── VisualEffects (Node3D) ← Partículas y efectos ├── TrailParticles (GPUParticles3D) └── ImpactDecal (Decal)

3. Sistema de Estados

3.1 Diagrama de Transiciones

┌─────────────┐ │ POSITIONING │ ← Estado inicial │ │ │ • Mover A/D │ │ • Click → │ └──────┬──────┘ │ Click izquierdo ▼ ┌─────────────┐ │ AIMING │ │ │ │ • Drag back │ │ • Push fwd │ │ • Soltar → │ └──────┬──────┘ │ Mouse UP (power ≥ min) ▼ ┌─────────────┐ │ THROWING │ │ │ │ • Impulso │ │ • Curva │ └──────┬──────┘ │ physics_update (1 frame) ▼ ┌─────────────┐ │ ROLLING │ │ │ │ • Física │ │ • Vel < 0.05│ │ por 0.5s →│ └──────┬──────┘ │ Bocha detenida ▼ ┌─────────────┐ │ POSITIONING │ ← Ciclo reinicia └─────────────┘

3.2 Descripción de Cada Estado

EstadoResponsabilidadFísicaInput
POSITIONING Movimiento lateral con A/D dentro de límites. Click izquierdo inicia AIMING. Freeze = true A/D + Click
AIMING Detecta gesto de 2 fases: PULL_BACK → PUSH_FORWARD. Feedback visual en tiempo real. Freeze = true Mouse drag
THROWING Aplica impulso central y velocidad angular (curva). Dura 1 physics frame. Freeze = false → impulse Ninguno
ROLLING Monitorea velocidad. Cuando cae debajo del umbral por tiempo configurable, vuelve a POSITIONING. Freeze = false (libre) Ninguno

4. Mecánica de Lanzamiento (ThrowController)

4.1 Gesto de Dos Fases

Fase 1: PULL_BACK (Cargar) ════════════════════════════ Click en bocha │ ▼ ┌─────────┐ │ BOCHA │ ←── Arrastrar mouse HACIA ATRÁS └─────────┘ (offset crece, color → rojo) │ │ drag_distance aumenta ▼ Punto máximo alcanzado (max_pull_distance) Fase 2: PUSH_FORWARD (Apuntar/Soltar) ══════════════════════════════════════ │ ▼ ┌─────────┐ │ BOCHA │ ←── Mover mouse HACIA ADELANTE └─────────┘ (push_offset calcula fuerza y dirección) │ │ push_distance crece ▼ Mouse UP → RELEASED → Lanzamiento DIAGRAMA DE FLUJO COMPLETO: ════════════════════════════ IDLE │ │ Click sobre bocha (raycast valida) ▼ PULL_BACK ──────────────────────────────┐ │ │ │ drag_offset crece │ Suelta sin │ max_pull_distance se actualiza │ hacer pull-back │ │ (offset.x > 0) │ ¿Se acercó al punto │ → IDLE (ignora) │ de inicio > 0.15m? │ │ │ ▼ │ PUSH_FORWARD │ │ │ │ push_offset calcula │ │ dirección y potencia │ │ │ ▼ │ RELEASED (Mouse UP) ─────────────────────┘ │ ▼ get_throw_data() → { power, direction, curve }

4.2 Cálculo de Potencia

push_distance ───────────── = normalized (0.0 → 1.0) max_drag_distance (4.0m) power = lerpf(min_throw_power, max_throw_power, normalized) Ejemplo: min_throw_power = 3.0 max_throw_power = 15.0 push = 0m → power = 3.0 (mínimo) push = 2m → power = 9.0 (50%) push = 4m → power = 15.0 (máximo)

4.3 Cálculo de Dirección

direction = push_offset.normalized() push_offset = current_world_pos - push_start_pos +X = adelante (dirección de lanzamiento) +Z = derecha -Z = izquierda La curva se calcula del desvío lateral (eje Z) del push: curve = clamp(push_offset.z / max_drag_distance * sensitivity, -1.0, 1.0) Esto permite "efecto" lateral natural: si el push no es perfectamente recto, la bocha curva.

4.4 Prevención de Tiros hacia Atrás

Regla crítica: Si el usuario suelta durante PULL_BACK pero movió el mouse hacia adelante (drag_offset.x > 0), el lanzamiento se ignora y se resetea a IDLE. Esto evita que un movimiento accidental hacia adelante al hacer click resulte en un tiro hacia atrás.

5. Proyección 3D del Mouse

El ThrowController no trabaja con píxeles de pantalla. Proyecta la posición del mouse a un plano horizontal en el mundo 3D:

Cámara │ │ ray_origin + ray_dir * t ▼ ╱ ╱ ← Rayo desde cámara ╱ ┌───────────────────────── ← Plano horizontal (drag_plane_normal = UP) │ · ← hit_pos │ a la altura de la bocha │ ┌─────────┐ │ │ │ BOCHA │ │ │ └─────────┘ │ └─────────────────────────┘ Intersección rayo-plano: t = (normal · (plane_point - ray_origin)) / (normal · ray_dir) world_pos = ray_origin + ray_dir * t

6. Feedback Visual (BochaAnimator)

Durante PULL_BACK: ┌─────────┐ │ BOCHA │ → Mesh se desplaza hacia atrás (drag_offset) └─────────┘ Color: blanco → rojo (según progress) Flecha roja: indica dirección del arrastre Durante PUSH_FORWARD: ┌─────────┐ │ BOCHA │ → Mesh vuelve hacia adelante (-push_offset) └─────────┘ Color: se aclara (progress decrece) Flecha: dirección del push RELEASED / IDLE: ┌─────────┐ │ BOCHA │ → Mesh vuelve a posición original └─────────┘ Color: material original Flecha: oculta

7. Guía de Balanceo del Juego

Esta sección detalla cada parámetro ajustable, dónde se encuentra, qué afecta en la jugabilidad y qué valores recomendamos para diferentes estilos de juego.

7.1 Parámetros de Lanzamiento (bocha.gd)

ParámetroDefaultEfectoRango sugerido
min_throw_power 3.0 Fuerza mínima del lanzamiento. Si el push fue muy corto, la bocha sale con esta fuerza. 2.0 – 5.0
max_throw_power 15.0 Fuerza máxima con push completo. Determina qué tan lejos puede llegar la bocha. 10.0 – 25.0
lateral_speed 3.0 Velocidad de movimiento lateral (A/D) durante POSITIONING. 2.0 – 6.0
lateral_bounds (-2.0, 2.0) Límites min/max del movimiento lateral en eje Z. Define el ancho de la "zona de tiro". (-3.0, 3.0) para cancha ancha
throw_sensitivity 1.0 Multiplicador del efecto curva. Valores altos hacen que pequeños desvíos laterales generen mucha curva. 0.5 – 2.0

7.2 Parámetros del Gesto (ThrowController.gd)

ParámetroDefaultEfectoRango sugerido
min_drag_distance 0.1 Distancia mínima (metros) para considerar que hubo un pull-back real. Si es muy bajo, movimientos mínimos activan el lanzamiento. 0.05 – 0.3
max_drag_distance 4.0 Distancia máxima para potencia 100%. Define cuánto debe empujar el usuario para lograr el tiro más fuerte. Este es el parámetro más importante para la sensación de control. 2.0 – 6.0
direction_change_threshold 0.15 Cuánto debe "volver" el mouse hacia el punto de inicio para detectar el cambio PULL_BACK → PUSH_FORWARD. Bajo = transición rápida (puede ser accidental). Alto = requiere un retorno claro. 0.1 – 0.4

7.3 Parámetros de Física (bocha.tscn — PhysicsMaterial)

ParámetroDefaultEfectoRango sugerido
friction 0.3 Fricción de la bocha contra el suelo. Más alto = frena más rápido. Más bajo = la bocha desliza más lejos. 0.1 – 0.6
bounce 0.2 Rebote al chocar con otras bochas o la cancha. Más alto = más rebote. 0.0 – 0.5
Nota: Estos valores están en el PhysicsMaterial inline dentro de bocha.tscn, no en un archivo .tres separado. Para modificarlos, editá directamente la escena o creá un recurso externo.

7.4 Parámetros de Rolling (RollingState.gd)

ParámetroDefaultEfectoRango sugerido
stop_threshold 0.05 Velocidad por debajo de la cual se considera que la bocha "casi paró". Muy alto = la bocha se detiene prematuramente. Muy bajo = tarda demasiado en reconocer que paró. 0.02 – 0.1
required_stop_time 0.5 Tiempo (segundos) que la bocha debe mantenerse por debajo del threshold para considerar que paró completamente. 0.3 – 1.0
trail_min_speed 0.5 Velocidad mínima para mostrar el efecto de trail/partículas. 0.3 – 1.0

7.5 Parámetros Visuales (BochaAnimator.gd)

ParámetroDefaultEfectoRango sugerido
max_visual_offset 1.5 Distancia máxima que el mesh se separa de su posición original durante el aiming. No afecta la física, solo el feedback visual. 1.0 – 2.5

8. Perfiles de Jugabilidad Recomendados

Perfil: Realista (Bocce tradicional)

min_throw_power = 2.0 max_throw_power = 12.0 max_drag_distance = 5.0 ← Push largo para máxima potencia friction = 0.4 ← Frena relativamente rápido bounce = 0.15 ← Poco rebote lateral_speed = 2.5 ← Movimiento lateral lento direction_change_thr = 0.25 ← Requiere gesto claro throw_sensitivity = 0.8 ← Curva sutil

Perfil: Arcade (Más rápido y permisivo)

min_throw_power = 4.0 max_throw_power = 20.0 max_drag_distance = 2.5 ← Push corto para máxima potencia friction = 0.15 ← Desliza mucho bounce = 0.4 ← Rebote notable lateral_speed = 5.0 ← Movimiento lateral rápido direction_change_thr = 0.1 ← Transición rápida throw_sensitivity = 1.5 ← Curva exagerada

Perfil: Competitivo (Precisión y control)

min_throw_power = 3.0 max_throw_power = 15.0 max_drag_distance = 4.0 ← Balanceado friction = 0.3 ← Estándar bounce = 0.2 ← Estándar lateral_speed = 3.0 ← Estándar direction_change_thr = 0.15 ← Requiere intención clara throw_sensitivity = 1.0 ← Curva natural

11. Comunicación entre Componentes

Todos los componentes comparten referencia a bocha_root bocha_root ├── physics_body ← RigidBody3D (acceso directo a física) ├── state_machine ← Transiciones de estado ├── position_controller ← Movimiento lateral ├── throw_controller ← Datos del gesto ├── visual_effects ← Partículas y efectos └── bocha_animator ← Feedback visual Señales emitidas por Bocha.gd: ├── bocha_thrown(power, direction) → Externo (UI, GameManager) ├── bocha_stopped → Externo (turno siguiente) └── state_changed(new_state) → Debug, UI

12. Checklist de Balanceo

Cuando ajustes la jugabilidad, seguí este orden:

  1. Definir la cancha: Ajustá lateral_bounds según el ancho real de tu escena de cancha.
  2. Definir la distancia máxima de tiro: Ajustá max_throw_power para que la bocha llegue al final de la cancha con un push completo.
  3. Ajustar la sensibilidad del gesto: Si el lanzamiento se siente "muy sensible", subí max_drag_distance (requiere más movimiento del mouse). Si se siente "pesado", bajalo.
  4. Ajustar la fricción: Si la bocha frena muy rápido, bajá friction. Si no frena nunca, subilo.
  5. Ajustar la curva: Si los tiros curvan demasiado, bajá throw_sensitivity y curve_multiplier.
  6. Ajustar el rebote: Si las colisiones entre bochas son muy elásticas, bajá bounce.
  7. Pulir el gesto: Ajustá direction_change_threshold para que la transición PULL_BACK → PUSH_FORWARD se sienta natural.
Tip de testing: Usá la escena TestGian.tscn con DebugHUD.gd para ver en tiempo real: estado actual, velocidad, posición, potencia, curva y fase del gesto. Esto es esencial para calibrar los valores.
