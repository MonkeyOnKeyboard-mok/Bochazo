Throw System v6 look_at() steering · Segmentos · Tiro libre · Diagonal angosta

Problemas que resuelve v6

NEW 1 Steering con look_at() — Impulso + redirección progresiva

En vez de rotar el velocity vector directamente, la estrategia v6 es: impulso inicial en la dirección del aim, luego la bocha usa look_at() para orientarse hacia donde se mueve efectivamente, y un steering progresivo la redirige hacia la curva deseada. Esto produce movimientos más naturales y visibles.

Impulso inicial + look_at() + steering suave por frames

NEW 2 Ventana de dirección diagonal pequeña

La cancha es más larga que ancha. Los tiros diagonales deben ser sutiles. Se reduce max_spread_deg de 3° a 1.5° para que las diagonales sean precisas y no violentas.

NEW 3 Análisis de forma por segmentos (Parábola, Logaritmo, C)

El promedio de curvatura no basta para curvas progresivas. Se divide el gesto en 3 segmentos (inicio, medio, fin) y se analiza la pendiente de cada uno. Esto permite diferenciar:

  • Parábola: curva fuerte al inicio, se endereza al final
  • Logaritmo: recto al inicio, curva fuerte al final
  • C (semicírculo): curva en un sentido, luego curva en el sentido contrario

NEW 4 Tiro libre (Free Throw)

El usuario dibuja una trayectoria completa durante AIM. La bocha debe seguir una forma similar a lo dibujado. Se muestrean los puntos del gesto y se convierten en waypoints de steering. La bocha usa look_at() hacia el waypoint activo y steering para acercarse.

NEW 5 Knuckleball con dirección horizontal

AIM horizontal (→ o ←) genera knuckleball pero con dirección lateral. El wobble se mantiene pero el impulso inicial tiene componente Z hacia el lado dibujado.

Problemas que resolvía v5

FIX 1 No detecta tiros rectos

Todo gesto con curvatura baja se clasifica como knuckleball (wobble). Se agrega una tercer categoría: straight. Si |curve| < straight_threshold → la bola va recta, sin wobble ni curva.

solo wobble y curve → straight (sin efecto), wobble (oscilación), curve (curva controlada)

FIX 2 max_draw_points se agota muy rápido

Al mover despacio, los puntos se acumulan y se alcanza el límite antes de terminar el gesto. Ahora max_draw_points solo cuenta durante la fase AIM, no durante CHARGE.

FIX 3 Los puntos de carga no deberían contar para el dibujo

Durante la fase CHARGE, el dibujo es una línea vertical fija. Solo los puntos de la fase AIM forman el gesto de dirección/curvatura.

NEW 4 Fases: CHARGE vs AIM

El gesto se divide en dos fases:

  • CHARGE — El usuario arrastra hacia abajo. Línea vertical fija. Acumula potencia.
  • AIM — El usuario cambia de dirección (arriba o lateral). Línea libre. Calcula dirección y curvatura.

Transición: CHARGE → AIM cuando dy ≤ 0 después de haber bajado al menos min_charge_distance px.

NEW 5 Botón de reset en test

Botón en la escena de test para resetear la bola sin reiniciar la escena.

NEW 6 Colores por fase

La línea de debug muestra dos colores:

  • ■ Naranja durante CHARGE — indica que se está cargando potencia
  • ■ Cyan durante AIM — indica que se está apuntando/curvando

Tipos de Tiro por Draw (v6)

Cada tiro se define por la forma del dibujo durante la fase AIM. Todos comienzan con CHARGE (carga hacia abajo). La estrategia central es: impulso inicial + look_at() para orientación + steering progresivo para curvas.

1. Knuckleball (sin AIM)

Draw: solo CHARGE, soltar sin AIM

Soltar después de cargar sin mover hacia ningún lado. La bola va recta con wobble sutil aleatorio.

Lógica: was_straight = true → Impulso recto + ThrowSpin en modo WOBBLE con fase aleatoria. Sin steering.

Steering: Ninguno. Solo wobble rotacional.

2. Straight — Tiro recto sencillo

Draw: CHARGE + ↑ (recto hacia arriba)

Después de cargar, mover recto hacia arriba. La bola va recta sin ningún efecto.

Lógica: aim_x ≈ 0, curve ≈ 0 → Impulso en dirección forward. Sin steering.

Steering: Ninguno. La bocha mantiene su trajectory recta.

3. Diagonal — Tiro recto diagonal

Draw: CHARGE + ↗ o ↖ (diagonal recta)

Después de cargar, mover en diagonal recta. La bola va en esa dirección con ángulo sutil.

IMPORTANTE: La ventana de dirección debe ser PEQUEÑA. La cancha es más larga que ancha. max_spread_deg = 1.5° (reducido de 3°). Un movimiento diagonal amplio no debe producir un ángulo exagerado.

Lógica: aim_x moderado (0.2–0.5), curve ≈ 0 → STRAIGHT + dirección diagonal angosta.

Steering: Ninguno. Impulso inicial ya tiene la dirección diagonal.

4. Knuckleball horizontal

Draw: CHARGE + → (horizontal derecha) o CHARGE + ← (horizontal izquierda)

Después de cargar, mover horizontalmente sin subir. La bola va en esa dirección con wobble.

Lógica: |aim_x| > 0.7 → WOBBLE + impulso con componente Z lateral.

Steering: Ninguno. Wobble rotacional + dirección lateral del impulso.

5. Parábola (y = x²)

Draw: CHARGE + curva tipo parábola (curva fuerte al inicio, se endereza)

La bola se mueve con curva pronunciada al inicio del trayecto, luego se endereza y va más recta.

Lógica: Análisis por segmentos → segmento_inicio tiene curvatura alta, segmento_fin tiene curvatura baja.

Steering: look_at() hacia dirección del movimiento + steering fuerte al inicio que decae progresivamente. El steering se multiplica por un factor que disminuye con la distancia recorrida.

6. Logaritmo (log-like)

Draw: CHARGE + curva tipo log(x) (recto al inicio, curva al final)

La bola empieza recta y después toma curva hacia el lado marcado.

Lógica: Análisis por segmentos → segmento_inicio tiene curvatura baja, segmento_fin tiene curvatura alta.

Steering: look_at() + steering que aumenta progresivamente. Factor de steering crece con la distancia recorrida.

7. Semicírculo (C)

Draw: CHARGE + forma de C

La bola curva hacia un lado y después se curva hacia el lado contrario.

Lógica: Análisis por segmentos → segmento_inicio y segmento_fin tienen curvatura de signo opuesto.

Steering: look_at() + steering que cambia de signo a mitad del trayecto. Se divide el recorrido en dos mitades: primera mitad steering en un sentido, segunda mitad en el contrario.

8. Tiro libre (Free Throw)

Draw: CHARGE + dibujo libre arbitrario

La bocha sigue una forma similar a lo dibujado. El gesto se convierte en waypoints de steering.

Lógica: Se detecta cuando el gesto no encaja en las categorías anteriores (curva compleja, múltiples cambios de dirección). Se muestrean los aim_points y se convierten a coordenadas 3D como waypoints.

Steering: look_at() hacia el waypoint activo + steering hacia ese waypoint. Cuando la bocha se acerca al waypoint (distancia < umbral), se pasa al siguiente. El último waypoint es la dirección final.

Análisis por segmentos: Se divide aim_points en 3 segmentos iguales (inicio, medio, fin). Para cada segmento se calcula la curvatura promedio (lateral deviation). La relación entre las 3 curvaturas determina el tipo de tiro:
  • Parábola: |curv_inicio| > |curv_medio| > |curv_fin| (decreciente)
  • Logaritmo: |curv_inicio| < |curv_medio| < |curv_fin| (creciente)
  • C: curv_inicio y curv_fin tienen signo opuesto

Tipos de Tiro por Draw (v5 — referencia anterior)

Cada tiro se define por la forma del dibujo durante la fase AIM. Todos comienzan con CHARGE (carga hacia abajo).

1. Knuckleball (sin AIM)

Draw: solo CHARGE, soltar sin AIM

Soltar después de cargar sin mover hacia ningún lado. La bola va recta con wobble sutil aleatorio.

Lógica: was_straight = true → ThrowHandler fuerza WOBBLE.

2. Straight (AIM hacia arriba)

Draw: CHARGE + ↑ (recto hacia arriba)

Después de cargar, mover recto hacia arriba. La bola va recta sin ningún efecto.

Lógica: aim_x ≈ 0, curve ≈ 0 → STRAIGHT.

3. Diagonal derecha (y = x)

Draw: CHARGE + ↗ (diagonal recta arriba-derecha)

Después de cargar, mover en diagonal recta hacia arriba a la derecha. La bola va en esa dirección con un ángulo sutil (max_spread_deg = 3°).

Lógica: aim_x > 0 moderado, curve ≈ 0 → STRAIGHT + dirección diagonal.

4. Diagonal izquierda (y = -x)

Draw: CHARGE + ↖ (diagonal recta arriba-izquierda)

Idéntico al anterior pero simétrico hacia la izquierda.

Lógica: aim_x < 0 moderado, curve ≈ 0 → STRAIGHT + dirección diagonal.

5. Knuckleball horizontal

Draw: CHARGE + → (horizontal) o CHARGE + ← (horizontal)

Después de cargar, mover horizontalmente sin subir. La bola va en esa dirección con wobble. Si se movió a la derecha, knuckleball hacia la derecha; si a la izquierda, hacia la izquierda.

Lógica: |aim_x| > 0.7 → WOBBLE + dirección horizontal.

6. Parábola (y = x²)

Draw: CHARGE + curva tipo parábola hacia la derecha (o izquierda)

Dibujar una media parábola. Si es para x > 0 (derecha), la bola se mueve levemente hacia la derecha y luego toma un efecto más recto. Si es para x < 0, simétrico hacia la izquierda.

Lógica: curve significativo → CURVE con spin_curve que indica la dirección.

7. Logaritmo

Draw: CHARGE + curva tipo log(x)

Forma de logaritmo: primero recto, luego se curva hacia el lado marcado. La bola empieza recta y después toma curva.

Lógica: curve significativo → CURVE.

8. Semicírculo (C)

Draw: CHARGE + forma de C

Dibujar algo estilo C: la bola curva hacia un lado y después se curva hacia el lado contrario.

Lógica: curve alto → CURVE. La forma de C se promedia; para curvas progresivas se necesita análisis de segmentos (pendiente).

Parábola, Logaritmo y C requieren análisis de forma más allá del promedio de curvatura. Actualmente el sistema usa el promedio, lo que produce resultados aproximados. Para curvas progresivas, se necesitaría dividir el gesto en segmentos (pendiente).

extends Node
class_name ThrowSteering

@export var ball: RigidBody3D
@export var steer_force: float = 0.3
@export var wobble_speed: float = 8.0
@export var wobble_force: float = 0.12
@export var waypoint_threshold: float = 0.5
@export var look_at_smoothing: float = 5.0

enum Mode { STRAIGHT, WOBBLE, PARABOLA, LOGARITHM, C_CURVE, FREE_THROW }

var mode: Mode = Mode.STRAIGHT
var active: bool = false
var wobble_time: float = 0.0
var wobble_phase: float = 0.0
var waypoints: PackedVector3Array = []
var current_waypoint: int = 0
var travel_progress: float = 0.0
var start_pos: Vector3 = Vector3.ZERO
var total_distance: float = 0.0
var base_curve: float = 0.0

func _ready():
    pass

func start_throw(throw_mode: Mode, curve_value: float, aim_direction: Vector3, gesture_points: PackedVector2Array = []):
    mode = throw_mode
    active = true
    wobble_time = 0.0
    travel_progress = 0.0
    current_waypoint = 0
    base_curve = curve_value
    start_pos = ball.global_position
    
    # Calcular distancia total estimada
    var vel = ball.linear_velocity
    if vel.length() > 0.1:
        total_distance = vel.length() * 2.0  # Estimación rough
    
    # Para free throw, convertir gesture_points a waypoints 3D
    if mode == Mode.FREE_THROW and gesture_points.size() > 1:
        _build_waypoints(gesture_points, aim_direction)
    
    # Fase aleatoria para wobble
    if mode == Mode.WOBBLE:
        wobble_phase = randf_range(0.0, TAU)

func _build_waypoints(points_2d: PackedVector2Array, aim_dir: Vector3):
    waypoints.clear()
    var start = points_2d[0]
    var end = points_2d[points_2d.size() - 1]
    var gesture_length = (end - start).length()
    if gesture_length < 1.0: return
    
    for p in points_2d:
        var t = (p - start).length() / gesture_length
        var lateral = (p - start).dot(Vector2(-aim_dir.z, 0)) / gesture_length
        var wp = Vector3(
            t * total_distance,
            0,
            lateral * total_distance * 0.3  # Scale down lateral
        )
        waypoints.append(wp)

func _physics_process(delta):
    if not active: return
    
    var vel = ball.linear_velocity
    if vel.length() < 0.1:
        active = false
        return
    
    # look_at hacia donde se mueve
    var target_dir = vel.normalized()
    var target_pos = ball.global_position + target_dir * 5.0
    ball.global_position = ball.global_position  # No mover posición
    # Usar lerp para suavizar la rotación
    var current_basis = ball.global_transform.basis
    var target_transform = ball.global_transform.looking_at(target_pos, Vector3.UP)
    ball.global_transform.basis = current_basis.slerp(target_transform.basis, look_at_smoothing * delta)
    
    # Calcular progreso del viaje
    var traveled = (ball.global_position - start_pos).length()
    if total_distance > 0:
        travel_progress = clampf(traveled / total_distance, 0.0, 1.0)
    
    # Aplicar steering según modo
    var steer_lateral: float = 0.0
    
    match mode:
        Mode.WOBBLE:
            wobble_time += delta
            steer_lateral = sin(wobble_time * wobble_speed + wobble_phase) * wobble_force
        
        Mode.PARABOLA:
            # Curva fuerte al inicio, se endereza
            var factor = 1.0 - travel_progress
            steer_lateral = base_curve * steer_force * factor
        
        Mode.LOGARITHM:
            # Recto al inicio, curva al final
            var factor = travel_progress
            steer_lateral = base_curve * steer_force * factor
        
        Mode.C_CURVE:
            # Cambia de signo a mitad
            var sign = 1.0 if travel_progress < 0.5 else -1.0
            var factor = 1.0 - absf(travel_progress - 0.5) * 2.0  # Max en los extremos
            steer_lateral = base_curve * steer_force * sign * factor
        
        Mode.FREE_THROW:
            steer_lateral = _steer_to_waypoint()
    
    # Aplicar fuerza lateral
    if absf(steer_lateral) > 0.001:
        var right = ball.global_transform.basis.x
        ball.apply_central_force(right * steer_lateral)

func _steer_to_waypoint() -> float:
    if waypoints.size() == 0 or current_waypoint >= waypoints.size():
        active = false
        return 0.0
    
    var target = waypoints[current_waypoint]
    var to_target = target - (ball.global_position - start_pos)
    
    if to_target.length() < waypoint_threshold:
        current_waypoint += 1
        if current_waypoint >= waypoints.size():
            active = false
            return 0.0
        target = waypoints[current_waypoint]
        to_target = target - (ball.global_position - start_pos)
    
    # Steering hacia el waypoint
    var right = ball.global_transform.basis.x
    return to_target.dot(right) * steer_force

Integración con GestureShapeAnalyzer

Para determinar el tipo de tiro (parábola, logaritmo, C), se necesita un nuevo componente que analice la forma del gesto por segmentos:

extends Node
class_name GestureShapeAnalyzer

@export var tracker: GestureTracker

signal shape_analyzed(shape_type: int, curve_value: float, waypoints: PackedVector2Array)

enum ShapeType { STRAIGHT, KNUCKLEBALL, DIAGONAL, PARABOLA, LOGARITHM, C_CURVE, FREE_THROW }

func analyze(aim_points: PackedVector2Array, was_straight: bool, aim_x: float) -> void:
    if was_straight:
        shape_analyzed.emit(ShapeType.KNUCKLEBALL, 0.0, aim_points)
        return
    
    if aim_points.size() < 6:
        shape_analyzed.emit(ShapeType.STRAIGHT, 0.0, aim_points)
        return
    
    # Dividir en 3 segmentos
    var third = aim_points.size() / 3
    var seg1 = aim_points.slice(0, third)
    var seg2 = aim_points.slice(third, third * 2)
    var seg3 = aim_points.slice(third * 2)
    
    var c1 = _segment_curve(seg1)
    var c2 = _segment_curve(seg2)
    var c3 = _segment_curve(seg3)
    
    var avg_curve = (c1 + c2 + c3) / 3.0
    
    # Clasificar por patrón de curvatura
    if absf(avg_curve) < 0.01:
        if absf(aim_x) > 0.7:
            shape_analyzed.emit(ShapeType.KNUCKLEBALL, aim_x * 0.3, aim_points)
        elif absf(aim_x) > 0.15:
            shape_analyzed.emit(ShapeType.DIAGONAL, 0.0, aim_points)
        else:
            shape_analyzed.emit(ShapeType.STRAIGHT, 0.0, aim_points)
    elif absf(avg_curve) < 0.2:
        shape_analyzed.emit(ShapeType.KNUCKLEBALL, avg_curve, aim_points)
    else:
        # Analizar patrón
        var shape = _classify_shape(c1, c2, c3, avg_curve)
        shape_analyzed.emit(shape, avg_curve, aim_points)

func _segment_curve(points: PackedVector2Array) -> float:
    if points.size() < 2: return 0.0
    var start = points[0]
    var end = points[points.size() - 1]
    var dir = (end - start).normalized()
    if dir.length() < 0.1: return 0.0
    var perp = Vector2(-dir.y, dir.x)
    var total = 0.0
    for p in points:
        total += (p - start).dot(perp)
    return total / points.size()

func _classify_shape(c1: float, c2: float, c3: float, avg: float) -> int:
    var abs1 = absf(c1)
    var abs2 = absf(c2)
    var abs3 = absf(c3)
    
    # C: signos opuestos entre inicio y fin
    if sign(c1) != sign(c3) and abs1 > 0.1 and abs3 > 0.1:
        return ShapeType.C_CURVE
    
    # Parábola: decreciente
    if abs1 > abs2 and abs2 > abs3:
        return ShapeType.PARABOLA
    
    # Logaritmo: creciente
    if abs1 < abs2 and abs2 < abs3:
        return ShapeType.LOGARITHM
    
    # Default: curve normal
    return ShapeType.FREE_THROW

        is_tracking = event.pressed
        if is_tracking:
            start_pos = event.position
            aim_points.clear()
            charge_distance = 0.0
            phase = Phase.CHARGE
            prev_pos = start_pos
            if charge_line: charge_line.clear_points()
            if aim_line: aim_line.clear_points()
            _add_charge_point(start_pos)
            gesture_started.emit(start_pos)
        else:
            var was_straight = (phase == Phase.CHARGE)
            gesture_ended.emit(aim_points, was_straight)

    elif is_tracking and event is InputEventMouseMotion:
        var pos = event.position
        var dy = pos.y - prev_pos.y

        if phase == Phase.CHARGE:
            _add_charge_point(pos)
            charge_distance += maxf(dy, 0.0)

            # Transición a AIM: dejó de bajar O llegó al tope de carga
            if (dy <= 0.0 and charge_distance >= min_charge_distance) or charge_distance >= max_charge_distance:
                phase = Phase.AIM
                if aim_line: aim_line.clear_points()
                aim_points.clear()
                aim_points.append(pos)
                if aim_line: aim_line.add_point(pos)
                gesture_phase_changed.emit(Phase.AIM)

        elif phase == Phase.AIM:
            if max_aim_points > 0 and aim_points.size() >= max_aim_points:
                is_tracking = false
                gesture_ended.emit(aim_points, false)
            else:
                aim_points.append(pos)
                if debug_enabled and aim_line:
                    aim_line.add_point(pos)

        gesture_dragging.emit(pos, start_pos)
        prev_pos = pos

func _add_charge_point(pos: Vector2):
    # Durante CHARGE, la línea es vertical: X = start.x, Y = current.y
    var vertical_pos = Vector2(start_pos.x, pos.y)
    if debug_enabled and charge_line:
        charge_line.add_point(vertical_pos)

    if not tracker: return
    tracker.gesture_ended.connect(_calc_curve)

func _calc_curve(aim_points: PackedVector2Array, was_straight: bool):
    if was_straight or aim_points.size() < 3:
        spin_calculated.emit(0.0)
        return

    var start = aim_points[0]
    var end = aim_points[aim_points.size() - 1]
    var dir = (end - start).normalized()
    if dir.length() < 0.1: dir = Vector2(0, 1)

    var perp = Vector2(-dir.y, dir.x)
    var total = 0.0

    for p in aim_points:
        var lateral = (p - start).dot(perp) / sensitivity
        total += clampf(lateral, -1.0, 1.0)

    spin_calculated.emit(clampf(total / aim_points.size(), -1.0, 1.0))

extends Node
class_name ThrowSpin

@export var ball: RigidBody3D
@export var spin_module: GestureSpin
@export var steer_force: float = 0.3
@export var straight_threshold: float = 0.01
@export var min_curve_threshold: float = 0.2
@export var wobble_speed: float = 8.0
@export var wobble_force: float = 0.12

enum Mode { STRAIGHT, WOBBLE, CURVE }

var spin_curve: float = 0.0
var mode: Mode = Mode.STRAIGHT
var active: bool = false
var wobble_time: float = 0.0
var wobble_phase: float = 0.0

func _ready():
    if spin_module:
        spin_module.spin_calculated.connect(_on_spin)

func set_knuckleball():
    mode = Mode.WOBBLE
    spin_curve = 0.0
    active = true
    wobble_time = 0.0
    wobble_phase = randf_range(0.0, TAU)

func _on_spin(curve: float):
    wobble_time = 0.0
    var abs_c = absf(curve)
    if abs_c < straight_threshold:
        mode = Mode.STRAIGHT
        spin_curve = 0.0
        active = false
    elif abs_c < min_curve_threshold:
        mode = Mode.WOBBLE
        spin_curve = 0.0
        active = true
        wobble_phase = randf_range(0.0, TAU)
    else:
        mode = Mode.CURVE
        spin_curve = curve
        active = true

func _physics_process(delta):
    if not active: return
    var vel = ball.linear_velocity
    if vel.length() < 0.1:
        active = false
        return

    match mode:
        Mode.WOBBLE:
            wobble_time += delta
            ball.linear_velocity = vel.rotated(Vector3.UP, sin(wobble_time * wobble_speed + wobble_phase) * wobble_force * delta)
        Mode.CURVE:
            ball.linear_velocity = vel.rotated(Vector3.UP, -spin_curve * steer_force * delta)
Knuckleball casuística: Cuando was_straight == true (solo se cargó, sin aim), ThrowHandler llama spin.set_knuckleball(). Cuando |aim_x| > 0.7 (aim horizontal), también fuerza WOBBLE con la dirección del aim.

var _pending_throw_data: Dictionary = {}

func _on_throw(aim_points: PackedVector2Array, was_straight: bool):
    if not power_bar: return
    var p = clampf(power_bar.value / 100.0, 0.0, 1.0)
    if p < min_power: return
    
    # Guardar datos para usar cuando shape_analyzer responda
    _pending_throw_data = {
        "power": p,
        "aim_points": aim_points,
        "was_straight": was_straight,
        "aim_x": direction.aim_x if direction else 0.0
    }
    
    # Pedir análisis de forma
    if shape_analyzer:
        shape_analyzer.analyze(aim_points, was_straight, _pending_throw_data.aim_x)
    else:
        _on_shape(GestureShapeAnalyzer.ShapeType.STRAIGHT, 0.0, aim_points)

func _on_shape(shape_type: int, curve_value: float, waypoints: PackedVector2Array):
    var data = _pending_throw_data
    var p = data.power
    var throw_type_name = _shape_type_name(shape_type)
    
    print("[ThrowHandler] %s | power: %d%% | curve: %.3f" % [throw_type_name, int(p * 100), curve_value])
    
    # Aplicar impulso inicial
    var throw_dir = aim.dir
    if shape_type == GestureShapeAnalyzer.ShapeType.KNUCKLEBALL and absf(data.aim_x) > 0.7:
        # Knuckleball horizontal: agregar componente lateral
        throw_dir = Vector3(aim.dir.x, aim.dir.y, data.aim_x * 0.3).normalized()
    
    ball.apply_central_impulse(throw_dir * p * max_force)
    
    # Iniciar steering
    if steering:
        var steer_mode = _shape_to_steer_mode(shape_type)
        steering.start_throw(steer_mode, curve_value, aim.dir, waypoints)

func _shape_type_name(shape_type: int) -> String:
    match shape_type:
        GestureShapeAnalyzer.ShapeType.STRAIGHT: return "straight"
        GestureShapeAnalyzer.ShapeType.KNUCKLEBALL: return "knuckleball"
        GestureShapeAnalyzer.ShapeType.DIAGONAL: return "diagonal"
        GestureShapeAnalyzer.ShapeType.PARABOLA: return "parabola"
        GestureShapeAnalyzer.ShapeType.LOGARITHM: return "logarithm"
        GestureShapeAnalyzer.ShapeType.C_CURVE: return "c_curve"
        GestureShapeAnalyzer.ShapeType.FREE_THROW: return "free_throw"
    return "unknown"

func _shape_to_steer_mode(shape_type: int) -> int:
    match shape_type:
        GestureShapeAnalyzer.ShapeType.STRAIGHT: return ThrowSteering.Mode.STRAIGHT
        GestureShapeAnalyzer.ShapeType.KNUCKLEBALL: return ThrowSteering.Mode.WOBBLE
        GestureShapeAnalyzer.ShapeType.DIAGONAL: return ThrowSteering.Mode.STRAIGHT
        GestureShapeAnalyzer.ShapeType.PARABOLA: return ThrowSteering.Mode.PARABOLA
        GestureShapeAnalyzer.ShapeType.LOGARITHM: return ThrowSteering.Mode.LOGARITHM
        GestureShapeAnalyzer.ShapeType.C_CURVE: return ThrowSteering.Mode.C_CURVE
        GestureShapeAnalyzer.ShapeType.FREE_THROW: return ThrowSteering.Mode.FREE_THROW
    return ThrowSteering.Mode.STRAIGHT
